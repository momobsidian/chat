🎯 WHAT HAPPENS WHEN SOMEONE COMES ONLINE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. User comes online in chat app
2. Firebase detects presence change
3. Server receives presence update
4. Server broadcasts to all connected clients:
   ✓ Android app (WebSocket)
   ✓ ESP32 device (WebSocket)
   ✓ Discord (Webhook)
   ✓ Telegram (Bot)
   ✓ Pushover (API)
   ✓ FCM (Android push)

5. ESP32 receives notification:
   ✓ OLED displays: User, Device, Time
   ✓ Buzzer plays: 4-tone melody
   ✓ LED blinks: 5 times rapidly
   ✓ Serial logs: Formatted output

STEP 1: Get Hardware
  □ ESP32 board
  □ OLED display (SSD1306, I2C, 4-pin)
  □ Buzzer (3-pin)
  □ Jumper wires
  □ USB cable

STEP 2: Install Software
  □ Arduino IDE
  □ ESP32 board support
  □ Libraries:
    - WebSocketsClient
    - ArduinoJson
    - Adafruit SSD1306
    - Adafruit GFX

STEP 3: Wire Components
  □ Follow ESP32-WIRING.md
  □ OLED: VCC, GND, SDA(21), SCL(22)
  □ Buzzer: VCC, GND, Signal(25)

STEP 4: Upload Sketch
  □ Open esp32-notification-client.ino
  □ Select board: ESP32 Dev Module
  □ Select port: Your COM port
  □ Click Upload

STEP 5: Monitor & Test
  □ Open Serial Monitor (115200 baud)
  □ Watch for WiFi connection
  □ Watch for WebSocket connection
  □ Test by coming online in chat
